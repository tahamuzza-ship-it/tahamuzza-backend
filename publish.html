Características Principales

Autenticación 2FA

Implementación TOTP con Google Authenticator, Authy y otras aplicaciones autenticadoras.

Acceso Restringido

Solo usuarios autenticados pueden acceder al portal. Control de acceso basado en roles.

Encriptación

Todos los datos están encriptados en tránsito y en reposo con los más altos estándares.

Códigos de Respaldo

10 códigos de respaldo para recuperación de cuenta si pierdes acceso al autenticador.

Gestión de Usuarios

Panel administrativo para gestionar usuarios, roles y permisos de acceso.

Dashboard Dinámico

Interfaz moderna con información en tiempo real y controles administrativos avanzados.

Características de Seguridad

JWT + Cookies HTTP-only
Tokens seguros con expiración de 7 días. Almacenamiento seguro de sesiones con flags HTTP-only.
2FA TOTP
Autenticación de dos factores basada en tiempo. Compatible con Google Authenticator y Authy.
Bcryptjs
Hash seguro de contraseñas con 10 rondas. Imposible recuperar contraseña original.
Códigos de Respaldo
10 códigos únicos para recuperación de cuenta. Se eliminan después de usarse.
Control de Acceso
Roles admin/usuario. Procedimientos protegidos para operaciones sensibles.
CORS Seguro
Configuración CORS restrictiva. Validación de origen en todas las solicitudes.

Instalación y Uso

1. Clonar o descargar el proyecto

git clone <repositorio> cd tahamuzza-backend

2. Instalar dependencias

npm install

3. Inicializar la base de datos

node init-db.js

4. Iniciar el servidor

node server.js

5. Acceder al portal

http://localhost:3000

Endpoints de API

Método Endpoint Descripción Autenticación
POST /api/auth/register Registrar nuevo usuario No
POST /api/auth/login Iniciar sesión No
POST /api/auth/verify-2fa Verificar código 2FA No
POST /api/auth/setup-2fa Generar QR para 2FA Sí
POST /api/auth/confirm-2fa Confirmar configuración 2FA Sí
POST /api/auth/disable-2fa Deshabilitar 2FA Sí
GET /api/auth/me Obtener usuario actual Sí
POST /api/auth/logout Cerrar sesión No
GET /api/users Listar usuarios (admin) Sí (Admin)
GET /api/users/:id Obtener usuario por ID Sí (Admin)
